DETALLES TÉCNICOS - BUG DE CARGA MÚLTIPLE
================================================================================

PROBLEMA:
En FileUpload.jsx línea 114, al mapear resultados se usa:
  const resultMap = new Map(rawResults.map((r) => [r.filename, r]));

Si dos archivos tienen el mismo nombre, el segundo sobrescribe el primero en el Map.

SOLUCIÓN REQUERIDA:
1. Frontend genera clientFileId único por archivo (uuid4 o Date.now() + random)
2. Frontend envía clientFileId en FormData junto con file
3. Backend recibe clientFileId, lo almacena y lo retorna en respuesta
4. Frontend usa clientFileId como clave del Map en lugar de filename

FLUJO CORREGIDO:
  FileUpload (genera clientFileId por archivo)
    → analysis.js (envía clientFileId en FormData)
    → uploadAndAnalyzeMultiple (backend recibe y retorna clientFileId)
    → FileUpload (mapea con Map.set(clientFileId, result))
