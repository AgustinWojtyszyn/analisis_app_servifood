Resumen de cambios
- Se simplifico la landing publica para que funcione como entrada institucional breve y legible.
- Se elimino la grilla extensa de tarjetas administrativas.
- Se redujo el contenido a cinco secciones: hero, beneficios, usos por perfil, acceso automatico y CTA con footer.
- Se mantuvieron unicamente los botones publicos Ingresar y Solicitar registro.
- Las columnas de uso son informativas y no permiten elegir rol.

Decisiones visuales
- Se mantuvo el fondo azul corporativo con gradientes sobrios.
- Se mantuvo el logo existente de ServiFood.
- Se mantuvo naranja como color principal de accion.
- Se hizo mas visible el boton Solicitar registro sin competir con Ingresar.

- /login responde HTTP 200 en Vite.
- /register responde HTTP 200 en Vite.
- El boton Ingresar usa el callback onLogin existente, conectado en App.jsx a navigatePublic('/login').
- El boton Solicitar registro usa el callback onRegister existente, conectado en App.jsx a navigatePublic('/register').
- PublicLanding.jsx no contiene href, to=, rutas privadas ni enlaces a modulos internos.
- PublicLanding.jsx no contiene textos "Ingresar como colaborador" ni "Acceso interno".
- PublicLanding.jsx no consulta rol, usuario, Supabase ni profiles.
- No hay tarjetas individuales para cada modulo administrativo.
- Las listas de funcionalidades son informativas y no navegables.
